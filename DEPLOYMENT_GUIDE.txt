=========================================================================
Lofi Space Station — 프로덕션 서버 배포 및 실행 가이드북 (space.raondr.com)
=========================================================================

본 문서는 프로젝트를 깃허브(GitHub) 공개 저장소에 푸시한 이후, 
실제 운영 서버(Ubuntu VPS 등)에서 수급 및 구동하기 위한 스텝 바이 스텝 가이드라인입니다.

-------------------------------------------------------------------------
1. 필수 사전 설치 항목 (Prerequisites)
-------------------------------------------------------------------------
서버 환경에 아래의 핵심 도구들이 사전 설치되어 있어야 합니다.

1.1 Node.js (v18 이상 권장)
    - 설치 확인: node -v
1.2 ffmpeg (NASA HLS 비디오 스트림의 실시간 프레임 분석에 필수)
    - Ubuntu 설치: sudo apt-get update && sudo apt-get install -y ffmpeg
    - 설치 확인: ffmpeg -version
1.3 yt-dlp (유튜브 NASA 라이브 피드 HLS 주소의 실시간 동적 파싱에 필수)
    - Ubuntu 설치: sudo wget https://github.com/yt-dlp/yt-dlp/releases/latest/download/yt-dlp -O /usr/local/bin/yt-dlp && sudo chmod a+rx /usr/local/bin/yt-dlp
    - 설치 확인: yt-dlp --version
1.4 PM2 (Node.js 백엔드 무중단 가동 프로세스 매니저)
    - 설치: sudo npm install -g pm2

-------------------------------------------------------------------------
2. 서버로 소스코드 내려받기 (Deployment)
-------------------------------------------------------------------------
운영 서버의 터미널에서 깃 저장소를 클론합니다.

$ git clone <YOUR_GITHUB_REPOSITORY_URL> lofi
$ cd lofi

-------------------------------------------------------------------------
3. 환경 설정 파일 수동 구축 (.env.local)
-------------------------------------------------------------------------
보안을 위해 `.env.local` 환경 설정 파일은 깃 제외 대상(.gitignore)으로 설정되어 서버에 올라가지 않습니다.
따라서, 서버 내부의 `backend/` 폴더 진입 후 파일을 직접 새로 만들어 생성해 주어야 합니다.

$ cd backend
$ nano .env.local

[아래의 내용을 그대로 작성하고 저장합니다 (Ctrl + O -> Enter -> Ctrl + X)]
---------------------------------------------------------
# Lofi Space Station Backend Configuration
API_BACKEND_HOST = "127.0.0.1"
API_BACKEND_PORT = 5000       # 배포 표준 포트 (로컬 개발/테스트 시에는 5001 권장)
ANALYSIS_INTERVAL_MS = 15000
---------------------------------------------------------

---------------------------------------------------------
4. 대용량 Lofi 음원 및 NASA 이미지 자동 수급 (Crucial)
---------------------------------------------------------
용량이 큰 음악 압축 파일(.zip)과 NASA 우주 이미지 자산은 깃 저장소에 업로드되지 않으므로, 
백엔드 최초 구동 전에 서버용 자동 다운로드 스크립트를 수동으로 1회 가동하여 미디어 풀을 즉시 구축해야 합니다.

$ cd backend
$ npm install  (백엔드 의존성 모듈 설치)

# 4.1 166곡 Lofi 음원 다운로드 (529MB)
$ node download-lofi.js
(529MB 압축 파일을 깃허브 릴리즈 허브에서 자동으로 스캔/다운로드하여 
 backend/media/music/ 경로 아래에 166곡의 음원을 자동으로 압축 해제해 줍니다.)

# 4.2 고화질 NASA 우주 이미지 다운로드 (슬라이드쇼용)
$ node download-nasa-images.js
(슬라이드쇼에 사용할 고화질 우주 WebP 이미지 31장을 수급하여 
 프론트엔드의 public/assets/nasa/ 디렉토리에 자동으로 셋업 완료해 줍니다.)
----------------------------------------------------------
5. 프론트엔드 빌드 (Build Frontend Assets)
---------------------------------------------------------
클라이언트 정적 빌드 파일을 생성합니다.

$ cd ../frontend
$ npm install  (프론트엔드 의존성 모듈 설치)
$ npm run build

(빌드가 완료되면 frontend/dist/ 디렉토리에 정적 빌드 HTML/JS/CSS 결과물과 함께 
 robots.txt, sitemap.xml 자산이 정상 생성됩니다.)

---------------------------------------------------------
6. 서버 백엔드 서비스 백그라운드 가동 (PM2 실행)
---------------------------------------------------------
PM2를 가동하여, 24시간 동안 실시간 NASA 카메라 분석 및 웹소켓 데이터 브로드캐스트가 무중단 가동되도록 설정합니다.

$ cd ../backend
$ pm2 start server.js --name "lofi-backend"
$ pm2 save
$ pm2 startup

---------------------------------------------------------
7. Nginx 리버스 프록시 및 SSL 웹 서버 연동 (space.raondr.com)
---------------------------------------------------------
일반 대중이 80(HTTP)/443(HTTPS) 포트를 통해 space.raondr.com 주소로 바로 진입하여 
프론트엔드 정적 파일과 백엔드 WebSocket API 포트(5000)를 통합 경유하도록 Nginx를 리버스 프록시로 매핑합니다.

7.1 Nginx 설정 구성 (/etc/nginx/sites-available/space.raondr.com)
---------------------------------------------------------
server {
    server_name space.raondr.com;

    # 1. 프론트엔드 정적 빌드 배포 경로 지정
    root /path/to/lofi/frontend/dist;
    index index.html;

    location / {
        try_files $uri $uri/ /index.html;
    }

    # 2. 백엔드 REST API 프록시 매핑
    location /api/ {
        proxy_pass http://127.0.0.1:5000/api/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection 'upgrade';
        proxy_set_header Host $host;
        proxy_cache_bypass $http_upgrade;
    }

    # 3. 백엔드 정적 음악 리버스 프록시 매핑 (Range 헤더 지원용)
    location /music/ {
        proxy_pass http://127.0.0.1:5000/music/;
        proxy_http_version 1.1;
        proxy_set_header Host $host;
        proxy_set_header Range $http_range;
        proxy_set_header If-Range $http_if_range;
        proxy_cache_bypass $http_upgrade;
    }

    # 4. 백엔드 실시간 WebSocket 통신 프록시 매핑
    location /ws/ {
        proxy_pass http://127.0.0.1:5000/ws/;
        proxy_http_version 1.1;
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "Upgrade";
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }
}
---------------------------------------------------------

7.2 Certbot을 통한 Let's Encrypt 무료 SSL(HTTPS) 인증서 설치
$ sudo ln -s /etc/nginx/sites-available/space.raondr.com /etc/nginx/sites-enabled/
$ sudo nginx -t
$ sudo systemctl restart nginx
$ sudo certbot --nginx -d space.raondr.com

위 설정을 완결하시면, space.raondr.com으로 접근 시 안전한 HTTPS 프로토콜(wss://)을 타고 
정상적으로 실시간 우주 로파이 음악 라디오 웹 서비스가 최적의 속도로 구동됩니다!
